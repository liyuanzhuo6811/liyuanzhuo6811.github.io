2025.11.3 CSP-S

T2 道路修复

好像是机房里为数不多没被卡常的。

考虑其实虽然 mmm 很大，但是实际上不在原图的最小生成树上的边都是完全没用的，因此可以只保留最小生成树上的边，把 mmm 变成了 nnn。

由于这个改造的代价很不好直接放到生成树计算上，而 kkk 又非常小，考虑枚举所有可能的改造集合 SSS。这样枚举以后就比较好做了，只需要把集合里的边和原来的边放在一起求最小生成树，就可以得到答案了。

然而这样的复杂度是 O(2knkα(n)log⁡nk)O(2^knk \alpha(n)\log nk)O(2knkα(n)lognk) 的，有点悬。于是考虑把一个集合 SSS 的某一个元素（记为 xxx）剔除，首先求出改造集合为 S\{x}S \backslash\{x\}S\{x} 的最小生成树，再把 xxx 的边都连上，复杂度就压掉了一个 kkk，变成了 O(2k(n+k)log⁡(n+k))O(2^k(n + k) \log (n + k))O(2k(n+k)log(n+k))。

场上就想到这个，就连山外的破电脑都只跑了1.2s，应该不会被卡了吧？不过其实还可以更优，考虑如果给每个乡村的边集都提前排序，在合并集合的时候就可以做到 O(n)O(n)O(n) 合并边的集合。这样的话复杂度就变成了 O(2k(n+k))O(2^k (n + k))O(2k(n+k))。可能是做的时间有点太长了，场上比较着急就没往下继续想。

T3 谐音替换

然后这个题就给创死了。

由于题目中的 s1=s2s_1=s_2s1​=s2​ 和 ∣t1∣≠∣t2∣|t_1| \neq |t_2|∣t1​∣=∣t2​∣ 的情况过于智障，直接默认两种情况都不存在。

场上想到的部分

首先那个一个串可以在不同位置替换达到同样的替换结果的情况是不存在的。证明起来比较简单，直接考察在 t1t_1t1​ 和 t2t_2t2​ 中第一个（最左边或者最右边都行）不相等的位置 iii，如果一种模式串可以匹配这个串，那么这种模式串在 t1/2,it_{1/2,i}t1/2,i​ 位置对应的也应当是第一个不相等的位置，因此是唯一的。

有这个结论以后就方便一些。考虑一组字符串的样子：

找出 t1t_1t1​ 和 t2t_2t2​ 的最长公共前缀和后缀，剩下的部分就是差异部分。发现如果某一组 sss 可以完成替换，则他的差异部分一定和 ttt 的差异部分完全相同。而 sss 的公共前缀应当是 ttt 的公共前缀的一个后缀，ttt 的后缀同理。

场下的考虑方向

到这里就肯定能想到用 AC 自动机搞一下吧，场上认为由于是 CSP 所以不应该用这个（当然也是不会），就给他跳过了。实际上是可以做的，把一组 s1s_1s1​ 和 s2s_2s2​ 表示成 pre#s1#s2#suf 的形式，其中 pre 表示公共前缀，suf 表示公共后缀，s1 和 s2 表示两个串的差异部分，中间用一个 # 做特殊字符就能直接匹配了。所以这就是个板子题啊，现场学了一个 AC 自动机然后五分钟就秒了。

不过理论上确实不需要用 AC 自动机。对的对的可以用后缀数组。

考虑首先前后要完全相同这个限制可以用哈希分类，不同类之间肯定不会产生答案，所以不管干什么都可以把哈希相同的单独干，这样就不存在差异部分相同这个限制了，只需要考虑前缀和后缀的关系。

然后想不下去了。前后缀同时满足的限制似乎有点过于严苛了。

题解

考虑如果 xxx 是 yyy 的前缀，那么 yyy 的终点一定在 xxx 的子树内，那么子树对应的就是连续的一段 DFS 序。那么对前后缀分别开一个 Trie（前缀需要先反转再往 Trie 里面插），对于每一组 ttt 就只需要找他的前缀和后缀被多少个 sss 的前后缀包含，这就相当于统计 ttt 的公共前缀在 Trie 上的 DFS 序被包含在多少组 sss 的公共前缀在 Trie 上的子树对应的 DFS 序区间，并且后缀也满足类似的限制。这样就转化成了一个点被包含在多少个矩形里面，也就能在 O(nlog⁡L)O(n \log L)O(nlogL) 的复杂度内求解了。

所以为啥考前几个星期都没学 AC 自动机啊……大凡我会了下午看见这个不就秒了。

2025.11.5 模拟赛 [USACO12FEB] Cow Coupons G

没学过反悔贪心，场上完全是在乱想，直接跳过自己想到的部分。

题解

建三个堆，前两个堆存储 cic_ici​ 和 pip_ipi​，而第三个堆暂时不管。

假设现在已经没有可用的优惠券了，又想使用一张优惠券，就需要撤销一张之前使用的优惠劵。撤销一张优惠劵的代价是 pj−cjp_j - c_jpj​−cj​，而撤销的时候显然是撤销代价最小的那个。这样的的话需要动态记录所有已经使用优惠劵的东西里面的最小代价，因此第三个堆就存储所有已经使用过优惠劵的撤销代价。而最开始可以给第三个堆里面塞上一堆 000 表示没有代价，这样就把有可用优惠劵的情况转化成没有优惠劵的情况。

然后做法就是每次查询第一个堆的堆顶元素和第二、三个堆的堆顶元素的和。比较这两个值，前者表示用原价购买一头牛，后者表示用优惠劵并且撤销一个之前的优惠劵。如果前者更优是没什么问题的，而后者更优的话就需要再在第三个堆里面加上这个元素的撤销代价。

关于“为什么不需要撤销原来选的奶牛”，可以考虑分类讨论：

  1. 撤销一头原价的奶牛。由于他是从堆里面被取出来的，证明无论是撤销原来的优惠劵并用新优惠劵买奶牛还是原价买其它的奶牛肯定都不如这个优，因此这种情况不存在。
  2. 撤销一头使用过优惠劵的奶牛。如果要这样的话，可以节省一张优惠劵，但是首先原价的奶牛不会和他抢这张优惠劵，而另一个使用优惠劵的奶牛又肯定比这个贵（否则他肯定会先于这头奶牛被选），所以也不优。

这样的话好像这个证明就问题不大了。

2025.11.6 模拟赛 [HAOI2010] 订货

模拟赛放的是加强版，所有的数据范围都被开到了 2×1062 \times 10^62×106。

自己想到的部分

方向有点问题。场上一直在尝试把所有的货弄成一些区间，考虑从某些地方往这里转移之类的东西。

然后就发现这样根本没法做，感觉这种东西做起来实在是太困难了，所以肯定是要从前往后在时间上扫。然而并不知道该记录什么。

题解

前面的货物在储存的时候会产生额外的代价。于是可以把每个货物的代价变成 di−i×md_i - i \times mdi​−i×m 然后相对代价就被确定了。这样的话当扫到某个位置 iii 的时候，考虑首先记下当前位置的代价（可以购买无限个），再把之前代价最小的 SSS 个物品记录下来，然后选择最小的 UiU_iUi​ 个。那么更新代价最小的 SSS 个物品，也就是说把两部分代价拼起来，去掉其中较大的一些使得只剩下 SSS 个物品。

这样的话直接扫一遍就可以了。具体实现的时候直接暴力删除会炸，然而发现这个过程可以简化，也就是说对于同一个位置购买的东西合并到一起，这样每个位置就只会加入并删除一次。可以用堆、平衡树之类的东西 log⁡\loglog 维护，当然也可以用单调队列 O(n)O(n)O(n) 维护。

第一部分：前置知识

P11714 [清华集训 2014] 主旋律

这个题的思路是省选那个题的一个真子集。

首先强连通这玩意明显难以刻画，因此考虑正难则反，也就是解决掉所有不是强连通图的情况。不是强连通又是有向图，那就是把强连通分量缩点，会得到一个节点数大于 111 的 DAG。

先不考虑缩点，考虑删成一个 DAG。DAG 最重要的点就是入度为 000 的点。删除这些点以后，剩下的图仍然是一个 DAG，同样会有一些入度为 000 的点。那么令 f(S)f(S)f(S) 表示集合 SSS 不强连通的方案，考虑枚举哪些节点成为了入度为 000 的点。

第二部分：C性质（所有边权相同）

由于所有的边权都是一样的，所以只要他还存在外向生成树就一定是最小外向生成树。

考虑最终的形态，一定是从一个强连通分量出发，能够到达所有的点。这实际上包含三部分内容：

  1. 有一个强连通分量。
  2. 从这个强连通分量出发能到达所有的点。
  3. 不存在能够回到这个强连通分量的边。

首先这样不会算漏，因为任意一种存在外向生成树的情况，都会包含从一个强连通分量出发、到达所有点的外向生成树。而算重也不存在，因为由于没有边能够回来，就算存在其它的强连通分量，他们也无法到达这个强连通分量，所以每种情况就只会被对应的强连通分量算一次。

然后分别计算这三个东西。强连通分量是好计算的，直接套用刚才的主旋律。而第二个则可以继续正难则反，考虑枚举从 SSS 出发有一个到不了的集合 TTT，则要想保证恰好只有这些点到不了，需要保证 S−TS - TS−T 是可达的，同时没有边能够到达 TTT。无法回来相对而言是最简单的，因为只需要删除所有从 U−SU - SU−S 向 SSS 的边即可。

这样的话，第二个限制考虑枚举所有的初始集合 RRR，记录 f(S)f(S)f(S) 表示从 RRR 出发，能够到达 SSS 这个集合的概率（当然 S⊆RS \subseteq RS⊆R）。然后按照刚才的思路推一下式子，得到：

f(S)=1−∑T⊆S,R∩T=∅f(S−T)2−E(S−T,T) f(S) = 1 - \sum_{T \subseteq S, R \cap T = \varnothing}f(S - T)2^{-E(S - T, T)} f(S)=1−T⊆S,R∩T=∅∑​f(S−T)2−E(S−T,T)

然而直接这么做的话是 O(4n)O(4^n)O(4n) 的，因为需要枚举所有的 RRR，对于这些 RRR 枚举超集，再枚举这些超集的子集。由于最终答案都是到 f(U)f(U)f(U) 的，可以考虑把状态的转移看作一张图，然后考虑这个 DP 的意义，就是从任意一个 RRR 的超集的地方出发（初始的代价为 111）走路，每次可以从一个子集的点走过来，乘上一个 2−E(S−T,T)2^{-E(S - T, T)}2−E(S−T,T) 的代价。然后求所有到 UUU 的路径和。这样的话就把所有路径都反过来，考虑从 UUU 出发到某些点的答案，然后对每一个起点都算一遍超集的贡献即可。

第三部分：正解

原来的生成树不需要考虑“最小”的约束，而现在有了这个限制。考虑 Kruskal 的过程，从边权小的边开始合并，则每种边权连完以后，不管选择了哪些边，都一定把这些点合并成了相同的连通块。对应到最小外向生成树上，就是这些边权给他连成了一个最小外向生成森林。

这样，假设已经连完了第 k−1k - 1k−1 小的边权，则要连大小为 kkk 的边的时候，实际上就是合并一些之前的树。

和之前一样分三部分，但是要稍微修改一下，就是原来的点变成了现在的连通块，因而存在的限制是需要满足连入的边要连到小生成树的强连通部分（至少要有到那里的边，否则根据“没有边能够回来”的限制这显然不是一棵外向生成树）。而这个强连通的部分是不确定的，所以需要根据上一轮的 DP 结果来分类这个部分。

强连通的限制依然用主旋律。但是没必要让所有小生成树都连通，只需要让这些小生成树的强连通部分都连在一起即可。于是就可以把每个强连通分量缩点，指定的根点可以接受有效的入边，所有点都可以输出出边。然后就可以跑主旋律了。

然后就是从这些节点可以到达所有的点。很重要的一点是连到某个生成树的强连通部分就一定能够到达所有节点，而反过来也是一样的，因此只需要考虑如何连到这些点就可以了，剩下的边不用管。

令 sc(S)sc(S)sc(S) 表示所有与 SSS 有交集的小生成树，be(S)be(S)be(S) 表示 sc(S)sc(S)sc(S) 中每棵树都以 SSS 为根的概率，则可以得到一个与上面的定义相似的 f(S)f(S)f(S) 表示以 RRR 为起点，SSS 恰好是所有的 sc(S)sc(S)sc(S) 的强连通部分，能够走到 sc(S)sc(S)sc(S) ——其实也就是走到 SSS。

这个转移还是照搬上面的东西：

f(S)=be(S)−∑T⊆S,sc(R)∩sc(T)=∅2−E(sc(S−T),T)f(S−T)be(T) f(S) = be(S) - \sum_{T \subseteq S, sc(R) \cap sc(T) = \varnothing}2^{-E(sc(S - T), T)}f(S - T)be(T) f(S)=be(S)−T⊆S,sc(R)∩sc(T)=∅∑​2−E(sc(S−T),T)f(S−T)be(T)

之所以 S−TS - TS−T 没有乘上 bebebe 是因为 fff 中就隐含了全部是强连通部分。这个转移说白了和上面一样，所有的点都能出边，只有根节点的入边有效。

这样再加上没有回来的边这个限制（这个限制过于简单），三部分合在一起就是最终答案。
