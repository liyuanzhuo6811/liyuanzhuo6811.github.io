Tricky

Cells Blocking

首先特判，如果起终点不连通的话就肯定是炸了。

这种只能向两个方向移动的问题可以考虑把尽可能向下的路径和尽可能向右的路径求出来，所有的路径都是夹在这中间的。

还是先考虑特殊情况：如果路径相交，那么这个点就是必由之路，堵上以后可以任选。这个情况需要去重。

另外的情况下，不难发现一定是一个点堵在一条路径上，否则一定走这两条中的一条到达。

此时考虑枚举某一条路径上的点，然后计算有多少合法的第二个点。这里合法意味着这是去掉第一个点以后的必由之路。

因此，可以考虑求出堵住第一个点以后，新的最靠边的路径。这需要用到这种图的一个性质：对于每一条对角线 vk:i+j=kv_k:i+j=kvk​:i+j=k，一条路径的第 iii 个点一定在对角线 vi+1v_{i + 1}vi+1​ 上。

首先预处理出哪些点是可以从左上联通到右下的，检查枚举到的这个点所在的对角线。不难发现选择这条对角线上在枚举点右上方的第一个可以联通的点是最优的。证明是显然的。因此只需要找到这个点，然后向两个方向分别找出靠边的路径就可以了。

这样之后，不难发现必由之路就是新路径和另一条路径的相交部分。

DS

Giant Penguin

首先考虑 k=0k=0k=0 也就是树的情况。这种情况下可以用点分树简单做到 O(nlog⁡n)O(n \log n)O(nlogn)。

问题是现在有一些环。考虑搞出来一个生成树，然后观察一下性质：由于环很少，每次找到重心并删掉以后其实只能够有 kkk 条边让两棵子树相邻（不是任意两棵有 kkk 条，而是总共有 kkk 条）。因此对于每条这样连接了两个子树的边，可以直接任选一个端点删掉。这样，不妨将这些端点和重心放在一起，定义这最多 k+1k + 1k+1 个点为这一层的关键点。在建树的时候，顺道跑几遍 BFS 把它们到子树任意顶点的最短距离计算出来。

考虑这样如何进行修改和查询。从一个节点向上，总共只会经过 O(klog⁡n)O(k \log n)O(klogn) 个关键点。因此对于一次操作，只需要更新它的所有祖先关键点上的信息。而对于一个查询也是同理。

感觉正确性需要考虑一下，不过模拟几种情况以后也不难论证。

DP

Edit Distance Yet Again

好好玩的题。

正常做编辑距离是 O(nm)O(nm)O(nm) 的，显然无法接受。

考虑到其实虽然暴力的状态数看起来很多，但是实际上由于存在 kkk 的限制，如果两个匹配差的太远就一定寄了，所以有用的状态没多少。因此就可以设 f(i,j)f(i, j)f(i,j) 表示 sss 匹配到了第 iii 位，ttt 匹配到了第 i+ji + ji+j 位的最小操作次数。

但是这样还是有点爆炸。考虑到其实最小操作次数是 O(k)O(k)O(k) 的，因此可以交换状态，也就是说记 f(i,j)f(i, j)f(i,j) 表示两个串的匹配长度距离位 jjj，操作次数位 iii 能达到的 sss 的最长前缀。这样状态的复杂度就成了 O(k2)O(k^2)O(k2) 了，看起来非常优秀。

然后就是转移的问题了。这个转移相对来说是简单的，只需要考虑三种情况：

  • sss 删掉一个字符。这对应 f(i,j)←f(i−1,j−1)+1f(i, j) \leftarrow f(i - 1, j - 1) + 1f(i,j)←f(i−1,j−1)+1。
  • sss 插入一个字符，相当于 ttt 删掉一个字符。这对应 f(i,j)←f(i−1,j+1)f(i, j) \leftarrow f(i - 1, j + 1)f(i,j)←f(i−1,j+1)。
  • 修改一个字符。这对应 f(i,j)←f(i−1,j)+1f(i, j) \leftarrow f(i - 1, j) + 1f(i,j)←f(i−1,j)+1。

然后这做完了吗？其实并没有。考虑到有可能当前位置其实已经完全匹配的，这个时候应该直接跳过它们而不是进行上面三种操作中的任何一种。因此，在完成一次这样的操作以后，还要查找 sss 和 ttt 对应位置的 LCP，将它们全都跳过。这可以二分哈希做。

写了一下，细节好像还挺多的。Link

Counting

Cactus

从简单的情况考虑。

如果图是个链的话，显然答案就是 k⋅(k−1)n−1k \cdot (k - 1)^{n - 1}k⋅(k−1)n−1。

假如是个环的话，情况相对来说就要复杂一些。首先把环断掉，这样就变成了一条链，链的计数是容易的。考虑这样只会有端点重复，那么就相当于是把两个端点看做一个点，算少一个点的方案数量。

那么就可以用一个 DP 来计算这个东西：设 fif_ifi​ 表示有 iii 个点的环的方案数，有 fi=k⋅(k−1)i−1−fi−1f_i = k \cdot (k - 1)^{i - 1} - f_{i - 1}fi​=k⋅(k−1)i−1−fi−1​。

接下来考虑仙人掌怎么做。还是先取消掉不在环上的边的限制，这样就变成了所有的环都是独立计数的。

接下来就要考虑加上一条树边会导致什么不合法的情况。相当于是一边钦定一个颜色，另一边就不能再使用这个颜色了。根据对称性不难发现在一个分量中，一个点被染成 kkk 种颜色的方案是相同的。因此合并两个分量就相当于把两个独立的分量乘起来，再乘以 k−1k\frac {k - 1}kkk−1​。
