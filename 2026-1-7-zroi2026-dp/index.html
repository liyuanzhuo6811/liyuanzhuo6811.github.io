Tenzing and Random Operations

开局上期望还是有点强度太大了。

这个题可以考虑组合意义。如果用乘法原理和加法原理来解释这个问题的话，就是说每一步有 aia_iai​ 种操作方式，而一次后缀加法相当于给这个后缀上的所有东西都加上 vvv 种操作方式，问总共有多少种操作方式。

尽管可以加很多次，但是不难发现其实有用的就是 nnn 次操作，因此记录 fi,jf_{i, j}fi,j​ 表示前 iii 步，其中用了 jjj 个加上的操作方式，剩下的操作用的是原有的 aaa 种操作方式。转移的时候，考虑 iii 是怎么操作的。

  1. 用了原有的 aia_iai​ 种方式中的一个，fi,j←fi−1,j⋅aif_{i, j} \leftarrow f_{i - 1, j} \cdot a_ifi,j​←fi−1,j​⋅ai​。
  2. 用了一个之前加上的操作方式，fi,j←fi−1,j⋅j⋅vf_{i, j} \leftarrow f_{i - 1, j} \cdot j \cdot vfi,j​←fi−1,j​⋅j⋅v。
  3. 用了一个新的操作，fi,j←fi−1,j−1⋅i⋅(m−j)⋅vf_{i, j} \leftarrow f_{i - 1, j - 1} \cdot i \cdot (m - j) \cdot vfi,j​←fi−1,j−1​⋅i⋅(m−j)⋅v。

为什么这样是对的？其实这个操作的核心在于让没有用到的加法暂时处于量子态，不去关心他的位置，而是等到用到的时候再从这些没有用到的里面选出一个，定下来他的位置。

[AGC022E] Median Replace

自动机。

判断一个字符串是否合法是自动机的老本行了。对于这个问题，用一个自动机从前往后扫，对于当前的最后三位，不难观察到一些性质：

  • 000000000 肯定可以直接消了，完全没用。
  • 111111111 肯定不会消去，因为两个挨着的 111 才能把 111 传播出去。
  • 如果是 101101101，显然应该把这个玩意删了，而如果是 110110110，可以留下来这个 000，因为可能可以和后面的 000 组成 000000000，效率更高。

经过这样的操作，最终一定是一长串 111 后面结合 0∼20 \sim 20∼2 个 000。如果 111 的长度比 222 还要长就一定可行了，剩下的情况分类讨论一下。

那么就可以用一个 DP 来进行计算：设 fi,a,bf_{i, a, b}fi,a,b​ 表示前 iii 个字符，压到最后剩下 aaa 个 111 和 bbb 个 000。状态好像还可以再压缩一下，反正就是能转移了。

【UNR #6】机器人表演

官方题解叽里咕噜说什么呢。

似乎好像也可以用状压并优化的视角来看，但是真的有必要吗？

考虑如果直接计数的话会很麻烦，因为最终可能会生成完全相同的两个序列，然后就数重了。

解决这个问题的方法就是让最终的一个形态只能对应一种构造方式。最简单的保证唯一性的方法就是贪心，只要设计一个合适的贪心策略，让这个终态按照这个策略反推出原来的状态就可以了。

具体到这个问题，可以考虑尽可能按照原串 SSS 匹配，只有当 SSS 没法匹配当前位置的时候，才放一个额外的括号上去。但是这样有一个问题：有可能会出现的情况是贪心导致某种括号匹配炸了，比如这个例子：SSS 是 (()，如果直接贪心匹配的话 (())(() 就直接炸了（当匹配到第四个位置的时候突然来了个右括号，而此时前三个都是贪心匹配的，因此就直接炸了，但是显然这个可以通过前面四个位置放额外括号来匹配。）

因此这个贪心如果仅仅这么设计会数漏，需要加上一个处理：如果发现匹配炸了，不能简单地掐死这个东西，而是应当撤回之前贪心匹配上的一段 SSS，把中间一段尝试改为用额外的括号匹配。具体来说仍然要满足最小修改的原则，找到前面最大的位置，使得这个位置的前缀和要比当前匹配的位置小，让这一段的匹配交给多余的括号。

这样就可以设计 DP，设 f(i,j,k)f(i, j, k)f(i,j,k) 表示现在构造了一个长度为 iii 的括号序列，其中 SSS 已经匹配到了第 jjj 位，额外的括号的前缀和（就是那个左括号 +1+1+1 右括号 −1-1−1 的东西）是 kkk。

转移的时候考虑这个位置选什么。如果选择了 Sj+1S_{j + 1}Sj+1​，就可以转移到 f(i+1,j+1,k)f(i + 1, j + 1, k)f(i+1,j+1,k)，否则可以转移到 f(i+1,j,k−Sj+1)f(i + 1, j, k - S_{j + 1})f(i+1,j,k−Sj+1​)。

但是问题是还有一种情况没有搞，也就是不够了的情况。这种情况只会在 Sj+1S_{j + 1}Sj+1​ 是左括号且 k=0k = 0k=0 的情况下出现。这个时候要跳回到前面的位置。跳到的位置可以直接预处理，则这种情况下可以转移到 f(i+1,prej,0)f(i + 1, pre_j, 0)f(i+1,prej​,0)。这样就可以放心大胆转移了。

空间似乎有点炸，不过第一维可以滚掉。时间复杂度 O(n3)O(n^3)O(n3)，空间复杂度 O(n2)O(n^2)O(n2)。

[CTS2019] 氪金手游

神人容斥啊……

首先如果所有的限制形成了一个外向树，即每个节点的权值比所有的子树内部的节点都小，那么这就是个大典题，大体就是考虑每个节点在其子树中第一个被选到的概率，乘起来就可以了。

现在问题是还有反向的边，这就搞得很难办。由于赋值没有相同的情况，不需要考虑神秘的相等情况，这个问题简单一些。发现如果让一个限制颠倒过来变成不满足这个限制，这条边就会反向。而如果不管这个限制，就相当于直接断掉这条边，使得问题从树变成森林。

这两个情况都非常好做。那么就可以容斥。首先答案是没有不满足限制的边的情况，因此可以看作是“不管所有的限制”去掉“存在至少一个不满足的限制”的样子，这就是非常经典的容斥形式，因此可以直接容斥去做。

但是似乎并没有到此为止，因为目前还需要记录很多东西，比如断了多少条边、WWW 的和是多少、这个连通块连了多少点之类的。因为最后还得套上容斥系数，就得记录更多东西。但是实际上不需要：在 DP 的时候带上容斥系数（也就是说记录“奇数个不合法的”减去“偶数个不合法的”的数量），这样就可以直接去掉一维，转移的时候如果要加上一个的话就直接给每个位置都乘以 −1-1−1。这样就可以设计出状态：f(u,k)f(u, k)f(u,k) 表示 uuu 子树，www 的和为 kkk 的概率，转移的话可以用树上背包。
