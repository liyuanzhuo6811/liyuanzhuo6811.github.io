后缀数组、后缀树与后缀自动机

发表于2026-03-26|更新于2026-04-11
|浏览量:|评论数:

后缀

设有一个字符串 SSS 长度为 nnn，则一个后缀为 S[i…n]S[i \dots n]S[i…n]。

相比于一个字符串的前缀，后缀显得更棘手一些：在求前缀的时候，只会在末端添加一个字符，字符串的形态相对而言变化比较小，可以用 Trie 之类的东西直接维护每个前缀，字典序之类的也比较显然；而后缀则是每次在前面插入一个字符，一次会把整个字符串平移一位，比较难搞。

但是后缀能够储存的信息是十分丰富的。所有的后缀的所有前缀就是这个字符串的所有子串，因此如果能够找到一种方式，在线性或者是单 log⁡\loglog 复杂度内维护所有的后缀，就可以快速的实现很多操作和查询。

首先一个 Naive 的想法是把所有的后缀插到一个 Trie 里面。这样确实能够达到目的，但是时间空间都比较爆炸。

另一个不那么 Naive 的想法是对所有的后缀进行排序。直接暴力显然还是不行。

后缀数组

后缀数组实现了对后缀进行排序。

明确一些约定：

  • saisa_isai​ 表示从小到大第 iii 个后缀。
  • rkirk_irki​ 表示 S[i…n]S[i\dots n]S[i…n] 的排名。

暴力的复杂度是 O(n2log⁡n)O(n^2 \log n)O(n2logn)。

考虑优化这个过程。整个排序的过程中最大的瓶颈在于对后缀进行比较。于是发现可以二分哈希优化这个过程，复杂度 O(nlog⁡2n)O(n \log^2 n)O(nlog2n)。

优化：倍增

对于字符串排序、比较这种问题，可以尝试的一个思路是首先找需要比较的两个串的第一个字符，然后再找下一个，以此类推。

如果数据随机的话不难发现这个算法的复杂度非常优秀，但是随便给个完全相等的字符串就卡掉了。

因此考虑对这个比较方式进行优化。发现可以每次倍增比较，即：首先比较每个后缀的第一个字符，也就是整个字符串所有长度为 111 的子串，然后比较前两个、前四个……依此类推。

这种做法的好处是，可以充分利用已经计算出的长度为 2k−12^{k - 1}2k−1 的子串的排名所提供的信息，用它们进行合并。每次合并是一个双关键字排序，也就是对于 iii 位置的后缀，相当于对 (rk(S[i…i+2k−1−1]),rk(S[i+2k−1…i+2k−1]))(rk(S[i\dots i+2^{k-1}-1]),rk(S[i+2^{k-1}\dots i+2^k-1]))(rk(S[i…i+2k−1−1]),rk(S[i+2k−1…i+2k−1])) 这个二元组进行排序。两个排名都是在上一轮排序中已知的。

优化：基数排序

对上面这个东西的排序，实际上依然是 O(nlog⁡2n)O(n \log^2 n)O(nlog2n) 的，只是常数相对小了一些。倍增的复杂度是消不掉的，要想再优化只有优化排序。基于比较的排序最快也只能达到 O(nlog⁡n)O(n \log n)O(nlogn)，所以只能考虑非比较的排序。桶排序不可行，因为值域是 O(n2)O(n^2)O(n2) 的，而对于这种多个关键字的排序，基数排序是个不错的选择。

在这里，考虑开一个辅助数组 tmptmptmp 表示 iii 后缀的第二排名关键字（也就是上文提到的 rk(S[i+2k−1…i+2k−1])rk(S[i+2^{k-1}\dots i+2^k-1])rk(S[i+2k−1…i+2k−1])）。这显然可以通过直接平移得到，当然也有一些比较有意思的常数优化。

有了 SA 以后似乎并不能够立刻做很多事情，因为这个信息还是太过于浓缩。一个重要的信息叫做 height 数组，这个信息维护了排名相邻的两个后缀的 LCPLCPLCP 的长度。

这个东西有个非常有意思的性质：heighti≥heighti−1−1height_i \ge height_{i-1}-1heighti​≥heighti−1​−1。证明的话考虑往后一位以后，原先的 LCP 一定还是 LCP。因此可以利用这个性质快速计算。

后缀树

对于将所有后缀放进一个 Trie 里面的思路，虽然直接硬做的话复杂度非常爆炸，但是这个东西可以优化。

首先，后缀树就是只保留根节点、终止节点和分叉节点的后缀 Trie。进一步地，如果只保留根节点、叶子节点和分叉节点（隐藏了一部分终止节点），就叫做隐式后缀树。

一个可行的求后缀树的方式是建出反串的 SAM，则 parent 树就是后缀树。

这样可以在字符串的前侧动态插入字符。但是如果反过来，需要在后方动态插入，则这个方法就行不通了。

Ukkonen 算法

这个算法很好玩。不过在介绍这个算法之前，需要先明确一些定义，并关注一些性质。

隐式后缀树的性质

  1. 隐式后缀树的每个叶子节点代表了只出现了一次的后缀。
  2. 一个分叉节点代表一个至少出现了两次的极长子串。注意：这并不代表所有的至少两次的极长子串都出现在分叉节点处，假如正好有一个后缀和另一个位置的子串在一起，这个位置并不会出现分叉。

active_point

设 kkk 为最小的不对应 SSS 的隐式后缀树中的叶子节点的后缀。定义 kkk 为 active_point。

显然 [1,k−1][1,k - 1][1,k−1] 中的后缀都只出现了一次。但是更为关键的性质是，[k,∣S∣][k, |S|][k,∣S∣] 中的后缀都出现了不止一次。这是因为考虑后面的后缀都是 active_point 位置的后缀的子串，这个位置的出现次数一定是不小于 active_point 位置的。

另一个性质是，随着在 SSS 的最后插入新字符，active_point 的位置是单调不降的。这个证明也是容易的，考虑如果它往后移了，那么就证明原先的那个位置也应该是出现了大于一次的。

后缀链接

由于后缀树本质上还是个 Trie，所以显然对于每个子串都能找到对应的节点（当然如果是隐式后缀树的话有些被藏起来了），这样定义 SSS 的后缀链接指向 SSS 去掉第一个字符的后缀。。

后缀链接倒是没什么性质，但是它的关键在于：由于隐式后缀树隐去了一些结束点，在构建后缀树的时候只需要考虑显示出来的点，这就要求考虑这个点的后缀链接是否被显示出来了。

事实上我们不关心叶子节点。对于分叉节点来说，显然如果这个节点对应的子串有 t+at+at+a 和 t+bt + bt+b 两种方案，那么这个节点的后缀也至少有这两种方案，所以后缀链接总是指向一个存在的节点而不是隐藏的。

有了这些性质以后，就可以着手构造后缀树了。

Ukknonen 算法的思路是：只构建 active_point 之前的后缀对应的后缀树。对于后面的暂时先留着，等到 active_point 右移的时候，再把新的叶子弄出来。

那么当新加入一个字符的时候，首先要更新原先的叶子节点。这些节点需要向后扩展一个字符。暴力插入就比较智障了，因此可以考虑每个叶子节点不再代表某个具体的位置，而是直接代表“到字符串的末端”。

另外，由于需要判断 active_point 是否发生变化，还需要判断原先 active_point 对应的后缀是否出现了两次。由于 active_point 对应后缀本身还没有被插入，所以只要能够在原来的后缀树上匹配，就证明 active_point 对应的后缀原来出现过，从而就一定出现了两次。

因此，当向后方插入新字符的时候，相当于将 active_point 最后又增加了一位，然后继续在树上匹配。这就会产生两种情况：

  • 匹配成功，这说明 active_point 没有发生变化，可以直接继续。
  • 匹配失败。这种情况下证明这个 active_point 位置的节点变成叶子节点了，需要在目前匹配的位置断开边（如果在边上的话），然后插入一个新的字符。同时需要将 active_point 右移，具体做法就是跳后缀链接，到达新的后缀。如果这个位置依然失配，就证明还得继续右移。重复这个操作，直到匹配或者到达根节点。

然而后缀链接仍然有两个待解决的问题。

第一个问题：如果当前在边上，根本没有后缀链接，怎么跳？

由于现在在边上，可以跳到当前边对应的上方节点的后缀链接，保持走的长度不变。但是这样可能会越过一些本来应该走的节点，需要走过这些节点。

第二个问题：后缀链接如何计算？

事实上，在这个过程中只考虑了显示出来的点的后缀链接，那么就不妨在新产生一个点的时候，将它的后缀链接确定出来。新产生一个点只有右移这一种情况，而需要的后缀链接是分叉处的。这个位置的后缀链接应该和它去掉一个字符的前缀的后缀链接是相符的，因此只要在往后跳的过程中，把新的后缀链接给找出来就可以了。

还有就是复杂度分析的问题。均摊的思路和 KMP 差不多：一次最多只会将匹配深度 +1，耗复杂度较大的分裂操作一次至少会将深度 -1，因此总的复杂度是 O(n)O(n)O(n) 的。

后缀自动机

之前看《算法竞赛》上一直没看懂，以为是自己太菜了，然后发现好像他啥也没讲。

SAM 是一个自动机，其在 O(n)O(n)O(n) 的时间和空间复杂度内，存储了 O(n2)O(n^2)O(n2) 量级的子串信息，是一个非常优秀的算法。

endpos

SAM 最核心的概念就是 endpos。其定义为一个字符串 ttt 在 SSS 中出现的右端点位置的集合。例如，加入 S=abcbacabbadS = \texttt {abcbacabbad}S=abcbacabbad，那么 endpos⁡(ab)={2,8}\operatorname{endpos}(\texttt{ab}) = \{2, 8\}endpos(ab)={2,8}。

不难将 endpos 合并为一些等价类，SAM 的每个节点相当于一个等价类。

显然这个定义有着许多优秀的性质。如果没有特殊说明，默认下面提到的字符串都至少在 SSS 中出现了一次。

  1. s1s_1s1​ 是 s2s_2s2​ 的后缀等价于 ∣s1∣≤∣s2∣|s_1| \le |s_2|∣s1​∣≤∣s2​∣ 且 endpos⁡(s2)⊆endpos⁡(s1)\operatorname{endpos}(s_2) \subseteq \operatorname{endpos}(s_1)endpos(s2​)⊆endpos(s1​)。显然每次出现 s2s_2s2​ 的时候一定会出现一次 s1s_1s1​，同时可以通过反证法证明充分性。
  2. 假设 ∣s1∣≤∣s2∣|s_1| \le |s_2|∣s1​∣≤∣s2​∣，如果 s1s_1s1​ 不是 s2s_2s2​ 的后缀，那么 endpos⁡(s2)∩endpos⁡(s1)=∅\operatorname{endpos}(s_2) \cap \operatorname{endpos}(s_1) = \varnothingendpos(s2​)∩endpos(s1​)=∅。这可以视作是 1 的推论。
  3. 一个 endpos 等价类包括所有是最长子串的后缀且长度大于最短子串的串。

后缀链接

这和后缀树的想法是差不多的，但是定义并不相同。

由于每次增量构造的时候需要增加 S[i…n]+cS[i\dots n] + cS[i…n]+c 的串，必然需要找到一种合适的模式，从当前节点跳到它的后缀，从而便于快速维护信息。

SAM 的后缀链接的定义需要考虑等价类的性质。举个例子，假如说一个字符串为 aabbabbcbba\texttt{aabbabbcbba}aabbabbcbba，那么对于字符串 aabb\texttt{aabb}aabb，其 endpos⁡={4}\operatorname{endpos} = \{4\}endpos={4}。然而，对于它的后缀 abb\texttt{abb}abb，其 endpos⁡={4,7}\operatorname{endpos} = \{4, 7\}endpos={4,7}。因此，每个等价类的后缀链接定义为最长的 ttt 使得 endpos⁡(t)≠endpos⁡(s)\operatorname{endpos}(t) \neq \operatorname{endpos}(s)endpos(t)=endpos(s) 的等价类节点。其性质是 shortest⁡(endpos⁡(s))=longest⁡(endpos⁡(t))+1\operatorname{shortest}(\operatorname{endpos}(s)) = \operatorname{longest}(\operatorname{endpos}(t)) + 1shortest(endpos(s))=longest(endpos(t))+1。

这个链接会一路连到根节点。

增量构造

假设目前我们已经构造好了 S[1…i]S[1\dots i]S[1…i] 的 SAM，现在要在后方扩展一个新的字符 ccc。

首先，显然 S[1…i+1]S[1\dots i + 1]S[1…i+1] 并不存在于原先的 SAM 中，因此一定会给他新建一个等价类。接下来的工作就是要更新其它位置的节点信息。

考虑不断沿着 S[1…i]S[1\dots i]S[1…i] 对应的等价类的节点的后缀链接向前跳。先考虑简单的情况，如果原先这个节点不存在 ccc 转移，那么这次插入不会造成任何影响，直接把它的 ccc 转移挂到新建节点上就好了。

否则考察在向前跳跃的过程中，第一个存在 ccc 转移的节点。设这个节点为 uuu，它转移到的节点为 xxx。

如果 longest⁡(x)=longest⁡(u)+1\operatorname{longest}(x) = \operatorname{longest}(u) + 1longest(x)=longest(u)+1，说明其实这个节点对应的字符串就是插入以后的一段后缀。那么这个时候其实依然好办，其实还是没有影响到节点，直接将后缀链接连到 xxx 上即可。

如果上式不成立，证明转移过去并不是当前的一个后缀。（这里是因为如果转移过去是后缀，那么在跳后缀链的时候，就会跳到这个后缀对应的 endpos 上）因此原先的等价类被破坏了，是当前串的后缀的那一部分的等价类上多了一个 i+1i + 1i+1，因此需要将原先的等价类拆分成两个。一部分储存 ≤longest⁡(u)+1\le \operatorname{longest}(u) + 1≤longest(u)+1 的节点，这部分的 endpos 增加；另一个储存 >longest⁡(u)+1> \operatorname{longest}(u) + 1>longest(u)+1 的节点，这部分节点的 endpos 不变。假设新复制的节点存储前者信息，记为 vvv。那么首先应当复制 xxx 的所有转移信息以及后缀链接。接下来，继续刚才后缀链上跳跃的过程：由于拆分节点的时候把当前后缀放到了 vvv 里面，因此如果后缀链上有转移到 xxx 的节点，应当将这些节点的转移修正为 vvv。同时，xxx 和新建节点的后缀链接都设置为 vvv。至此构造过程结束。

复杂度分析

从这个构造过程中不难看出，一次最多只能新建 222 个节点，因此节点数量不超过 2n−12n - 12n−1。同时这也就是其下限，不难构造出 abbbbbbbb 的字符串，其可以卡满 2n−12n - 12n−1 个等价类。

在构造过程中，不难发现其实每次只加入了常数个转移，因此转移数是 O(n)O(n)O(n) 的，更精确的分析表明其不会超过 3n−43n - 43n−4。这是一个非常优秀的复杂度，但是实际上如果用字典树方式存储所有转移的话空间复杂度是 O(n∣Σ∣)O(n |\Sigma|)O(n∣Σ∣) 的，也可以用 map 做到 O(nlog⁡∣Σ∣)O(n \log|\Sigma|)O(nlog∣Σ∣)，但是常数和查询的复杂度都要大。

ZROI2026二轮 动态规划
Cyclic Shift 把每个点向他后面第一个（包括循环位移的）大于他的点连边，这样形成了一棵树。 显然对于某一个点开始的序列，一定是从他到根的一条链。每个节点维护一个二元组 (val,len)(val, len)(val,len)，valvalval 表示这个节点的大小，lenlenlen 表示到下一个节点的距离。把这玩意压到一个哈希里面，找第一个不同的位置比较即可。
