复杂度

复杂度证明是线段树合并的核心。当然你也可以不证明直接瞎猜

一次对于满线段树合并的复杂度是严格的 O(nlog⁡n)O(n \log n)O(nlogn) 的。

但是实际上，这样的情况基本不存在。由于动态开点的存在，有些节点根本就没有建。这个时候，我们发现线段树的节点数量基本上和叶子节点是同一个量级的。而要想一通操作跑到叶子，必然需要两个节点的叶子同时存在。

因此得到推论：一次线段树合并的复杂度不超过两个线段树中较小的一个。

通常情况下，如果是在一棵树的每个节点统计信息并且合并儿子，那么这个推论就可以利用类似于 DSU on tree 的过程证明复杂度。

例题

Lomsat gelral

直接用线段树统计最大值和最大值的节点编号和。合并时如果两个节点有相同的最大值就把编号和相加，否则就保留较大的。

复杂度没问题。

Subtree Minimum Query

考虑线段树下标记距离，统计区间最小值。

但是这样的话子树信息没法往父亲合并（因为会偏移），所以改成下标记录深度。

统计比较简单。复杂度依然没问题。

Peaks

一眼 Kruskal 重构树

考虑离线的情况下，我们可以把所有的询问的 xxx 从小到大排序，处理询问的时候，把所有的边都加上，用并查集判断连通性。

发现这样的话，可以用一个权值线段树维护一个连通块内部的点权，连边的时候线段树合并。查询线段树二分即可。

[HEOI2016/TJOI2016] 排序

好有道理的解法。基本上就是一个大区间排序不好做，于是我们考虑颜色段均摊，记录一些单调的段（记录这个段是单增还是单减）。

这样其实还是不好做，因为排序的复杂度还是太高了。我们考虑可以对一些段开一个桶，然后对一个区间排序实际上就是把几个桶拍到一起。这样的话一次排序会把一堆段合并，同时分裂两个段。那么这些桶就都可以用权值线段树维护，然后进行线段树合并/分裂即可。

[JOISC 2020] 星座 3

首先这个小白船的形式就非常笛卡尔树，先把树建出来。

考虑树上 DP。我们记录 fu,hf_{u, h}fu,h​ 表示以 uuu 为根的子树，最高的星星高度为 hhh 的最小不自然度。

转移的时候，如果一个儿子的最高的星星高度低于这个节点的高度，转移的时候直接忽略就可以，因为这个不会对另一个儿子产生任何影响。反之如果有一边的高度是大于这个节点的高度，另一个就必须低于高度。

那么实际上转移就是：

fu,h=min⁡{fls,h+min⁡k≤hu{frs,k},frs,h+min⁡k≤hu{fls,k}}f_{u,h} = \min\{f_{ls, h} + \min_{k \le h_u}\{f_{rs,k}\}, f_{rs, h} + \min_{k \le h_u}\{f_{ls,k}\}\} fu,h​=min{fls,h​+k≤hu​min​{frs,k​},frs,h​+k≤hu​min​{fls,k​}}

上面的转移中，我们发现给每个 fuf_ufu​ 开一颗线段树，以 hhh 为下标，记录最小值。这样的话，min⁡k≤hu{frs,k}\min_{k \le h_u}\{f_{rs,k}\}mink≤hu​​{frs,k​} 可以直接 O(log⁡n)O(\log n)O(logn) 在左右儿子的线段树上查出来，相当于常数，那么可以用线段树合并来进行转移。

同时还要注意一种情况就是如果两个都比 huh_uhu​ 低并且 uuu 上方有星星的时候，还要单独处理这种情况。

问题出现我再告诉大家

题意

给一棵 111 为根的 nnn 个点的树和 {cn},{dn}\{c_n\}, \{d_n\}{cn​},{dn​}, 表示可以以 cuc_ucu​ 代价覆盖 uuu 子树中与 uuu 距离不超过 dud_udu​ 的点, 求每个点都被至少一个点覆盖的代价最小值。

题解

考虑 DP。设 fu,if_{u, i}fu,i​ 表示以 uuu 为根的子树，其中最上面 iii 层已经被覆盖的最小代价。转移非常方便：

fu,i=min⁡{∑fv,i−1,∑fv,du−1+cu}f_{u, i}=\min\{\sum f_{v, i - 1}, \sum f_{v, d_u - 1} + c_u\} fu,i​=min{∑fv,i−1​,∑fv,du​−1​+cu​}

发现后面的那个相当于常数，考虑前面这个东西需要平移，如果不用平移的话可以用线段树合并做。那就和上面那个题一样，用深度代替距离就可以去掉平移。

然后这个东西需要一个区间操作，如果需要证明复杂度的话需要优化，就是考虑正常的区间赋值如果 pushdown 会开一个新的节点，但是合并的时候如果一个节点实际上没有值，只是因为 pushdown 才开了一个新节点，那么合并的时候跳过即可。
