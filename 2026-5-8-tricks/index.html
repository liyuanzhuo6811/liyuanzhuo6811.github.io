这里整理一些常见的Tricks，并附上一些经典的题目。

由于整个整理完全没有任何头猪（看见什么整理什么），可能会比较乱，难度可能波动也比较大。

将网格图转化为二分图

这个通常用来解决在网格上覆盖或者匹配等问题。二分图相比于原图，某些情况下点数降下了一个数量级（因为只对关键点连边），同时具有一些非常优秀的性质，可以大大简化原问题的各种依赖限制。

下面是一些经典题目。

[problem:HDU-1045]

当然这个题你可以用 O(216)O(2^{16})O(216) 的做法通过，但是这里实际上有更好的方法。

考虑到如果没有那个墙的限制，就相当于是每个行列都连边，要求每行只能选一个，这不就是二分图最大匹配。

现在的问题是每一行被分开了，那么考虑实际上分开的部分是完全没有关系的，也就是说一堵墙墙相当于把一行拆成了两个点，他们分别向各自的列连边，整理一下就相当于是把每一行的每一部分向他们各自的列连边，二分图最大匹配即可，复杂度反正是多项式级别的。

[problem:HDU-1281]

和上面那个题的唯一区别就是这个题是空地不是墙。那么就不拆就和上一个题一样了。

牛客多校 Minimum grid

首先既然要求了最大值，那么就必然要把每一行和每一列找出一个位置放上这个最大值。唯一的例外在于如果有一个点的行和列的要求都是这个值，那么显然选择这个点染色是要优于分开染色的（就算不选这个位置，其他的值赋过来好像也并没有节约操作数量，因此只有用这个位置能够节约一个值），因此只需要找出总共能找出多少个这样的点。这就是一个非常经典的二分图最大匹配模型。

[problem:CSES-1709]

大典题，每次删除一行或者一列，问最少的删空的次数。

把整个图中每一个硬币都看作一条边的话，就相当于是求点覆盖数，这样跑二分图最大匹配即可。

[problem:洛谷-P12785]

【未完成】

这个题倒是没有重新建图，但是其实网格图本身就是个二分图。

考虑性质，相当于所有的叶子节点都在一侧，那么不妨设两个点集分别为 L,RL,RL,R，那么发现其实 ∣L∣=∣R∣|L| = |R|∣L∣=∣R∣ 的情况一定是无解的，因为没有叶子的一端的所有点度数一定都大于等于 222，因此边就炸了，所以得到只能让叶子在多的一侧。同理，这个事情对于没有叶子一侧的任意一个子集都是成立的。

Hall 定理告诉我们，在这种情况下一定存在 ∣L∣|L|∣L∣ 的匹配。因此考虑先把所有的匹配边都找出来，再连上一些边。

用拉格朗日插值加速 DP

思路就是，如果 fif_ifi​ 可以表示为一个多项式的形式 F(x)F(x)F(x)，那么就可以通过拉格朗日插值的方式，通过计算一些小项得到一个巨大的 fif_ifi​ 的答案。

[problem:CodeForces-995F]

设一个 f(u,j)f(u, j)f(u,j) 表示子树 uuu 中钦定 uuu 放 jjj 的方案数，加上前缀和优化可以做到 O(nD)O(nD)O(nD)。

但是这样无法通过，因此需要优化。由于 s(u,j)−s(u,j−1)=f(u,j)=∏s(v,j)s(u, j) - s(u, j - 1) = f(u, j) = \prod s(v, j)s(u,j)−s(u,j−1)=f(u,j)=∏s(v,j)，而叶子节点处是个一次的多项式，那么会发现实际上整个 s(u)s(u)s(u) 都是个不超过 sizusiz_usizu​ 此的多项式。

因此可以先跑出来 s(u,1…n)s(u, 1\dots n)s(u,1…n)，然后插值再把 DDD 带进去就可以得到答案。

[problem:洛谷-P4463]

首先考虑只需要数递增的数列，对于非递增的数列只需要乘个阶乘就可以了。

考虑怎么数，那其实就是上一个题的树变成链，做法是完全一样的。

[problem:洛谷-P3270]

虽然可以容斥推一大堆式子，但是好像直接插值会好想一些。

考虑设 f(i,j)f(i, j)f(i,j) 表示考虑了前 iii 门课，目前有 jjj 个同学被碾压的方案数。这个容易递推，只需要考虑前 i−1i - 1i−1 门课然后新的同学中有多少个上岸的即可。

然后唯一的问题是需要计算根据排名分配分数的得分方案数。如果直接暴力算的话需要枚举 B 得了多少分，比较麻烦，但是不难发现实际上这个东西的指数项不超过 nnn，于是代出来一些点就可以得到很大的值的答案了。

[problem:洛谷-P5469]

首先需要发现这是一道区间 DP 题。

然后需要一个关键的观察：考虑区间的最大值，这个最大值的左右两侧可以分别计算，恰好就可以将区间比较完美地分成两段，因为一定不存在跨过这个最大值的机器人。当然，如果存在多个最大值，需要选择最右边的那一个，因为向左移动的机器人可能跨越一个相等的值。

那么就可以设计一个状态 f(i,j,k)f(i, j, k)f(i,j,k) 表示区间 [i,j][i, j][i,j] 的最大值为 kkk 的填数方案数。转移的时候使用前缀和优化。然后发现这个区间 DP 不需要扫整个区间枚举最大值的位置，因为只能把他放在中间的地方才能让起点在最大值位置的机器人走的长度相同。

估摸一下，注意到每次最大值都只能放在正中间的位置，所以形成了一个类似于平衡树的结构，左右两边只会有常数个有效区间，所以总的有效区间只有 O(nlog⁡n)O(n \log n)O(nlogn) 个。这样一来，把所有的区间按照长度排序，转移的时候就可以设 f(i,k)f(i, k)f(i,k) 表示第 iii 个有效区间，区间内部的最大值为 kkk 的方案数，同样可以前缀和优化。这样一来，转移就变成了几个合法的左右区间转移过来并且乘起来。

那么对于前缀和，可以发现这仍然是一个 lll 次的多项式，因此可以考虑拉格朗日插值。不难发现如果带入的值连续，可以通过前后缀积优化，因此可以做到 O(n)O(n)O(n) 做一次求值，然后依次得到所有的答案。

总结

这应该算是当 DP 有一维很大而又没什么办法消去这一位的时候的一种非常有意思的做法。

消去大状态的方法实际上有很多，但是通常都是通过观察性质，把这一维变成可以计算的东西，而这种 Trick 则非常聪明地利用了 DP 方程的多项式性质，得到了一种在不需要对原来的 DP 进行任何修改的压缩方法。
