7.10 动态规划题目

计数类问题

CF1657E Star MST

首先，我们发现条件可以变为：图的最小生成树是以 111 为中心的菊花。根据 Kruskal 的性质，很明显对于 222 个节点 uuu 和 vvv，他们之间的边 u→vu \to vu→v 一定要大于 1→u1 \to u1→u 和 1→v1 \to v1→v。

然而这玩意没法按照 1∼n1 \sim n1∼n 的顺序 DP，因为你会发现如果枚举 1→x1 \to x1→x 的边的长度就不能转移，因此整个乱套了。

发现 u→vu \to vu→v 的边长实际上是要大于两点到 111 的边的 max⁡\maxmax。由于 max⁡\maxmax 不好转移，我们考虑能不能把这个 max⁡\maxmax 去掉。那么，如果我们按照边权来进行 DP 的话就好多了。

因此状态出来了：fi,jf_{i,j}fi,j​ 表示最大的边为 iii，总共有 jjj 个点的方案数。

初步写出一个转移：

fi,j=fi−1,k×(v−k+1)newedge×permf_{i, j}=f_{i - 1, k} \times (v - k + 1)^{\text{newedge}} \times \text{perm} fi,j​=fi−1,k​×(v−k+1)newedge×perm

其中，newedge 表示从原来的点连向新加的点的总数（他们都要大于等于 iii），perm 表示哪些点到 111 的边长为 iii。

不难发现，newedge 可以通过“大减小”的方式得到，就是 (j2)−(k2)\binom{j}{2}-\binom{k}{2}(2j​)−(2k​)，而另一个则更简单：(jk)\binom{j}{k}(kj​)——从 jjj 个中选 kkk 个

因此，转移也就出来了：

fi,j=fi−1,t×(v−i+1)(j2)−(k2)×(jk)f_{i, j}=f_{i-1, t}\times (v-i+1)^{\binom{j}{2}-\binom{k}{2}}\times \binom{j}{k} fi,j​=fi−1,t​×(v−i+1)(2j​)−(2k​)×(kj​)

[JOI Open 2016] 摩天大楼 / Skyscraper

这里，和上一个题的 max⁡\maxmax 一样，绝对值非常烦人，因此我们就直接拆掉他。

发现相邻两个数的贡献其实可以拆成一正一负，大的数的贡献是正的，小的数的贡献是负的。

这样就又变成了一个 max⁡\maxmax 问题。和上个题一样，直接按照值域的顺序 DP。

然后我们发现我们依然不会做……因为当我们按照值域做的时候，位置这一维就消失了，因此我们没法准确地得知现在的情况。然而，实际上两个数的具体的位置并不重要，因为我们在做的是排列，只要保证最后每个数都放进去就行。

这时，我们来看一下新加一个数的时候，会有哪些贡献情况。

  1. 这个数的一边有一个已经加上的数。由于我们按照值域的顺序做，它旁边的数显然比他小。而另一边没有，所以他的贡献一正一负，直接抵消。
  2. 两边都有数。两边都比他小，贡献为 2×Ai2 \times A_i2×Ai​。
  3. 两边都是空的微。两边都比他大，贡献为 −2×Ai-2 \times A_i−2×Ai​。

发现这些情况实际上只与当前区间的连续序列数量有关。因此开一个状态记录这个连续元素块的数量，就……做完了？

实际上，如果写出状态，它现在是这样的：

fi,j,k,0/1/2f_{i,j,k,0/1/2} fi,j,k,0/1/2​

分别表示当前的数的大小，连续元素的数量和它们的贡献和（最后一维用于判断端点）。前两维显然是满了，但是最后一维，由于贡献有正有负，我们不能在当前的贡献大于 LLL 的时候就扔掉它，于是这一维是 O(nV)O(nV)O(nV) 的。

那咋整？我们考虑取消“减”的操作，只保留加的操作。同时，我们每次增加一个数的时候，不止考虑这个数的位置，还考虑让每个连续段的两端都加上 Ai−Ai−1A_{i}-A_{i - 1}Ai​−Ai−1​。为什么呢？因为每一个“两端”的位置，未来一定会出现一个比它还大的数。这样的话，kkk 这一维就单增了，我们可以放心扔掉 ≥L\ge L≥L 的元素了。

这下真做完了。复杂度 O(n2V)O(n^2V)O(n2V)。

Large Heap

注意到这是在一个满二叉树上，最左端的一个节点和最右端的一个节点的问题。所以可以合并最左链和最右链，这样问题就变成了保证 bbb 在最下面的一个单增链上挂了一堆子树的形态的方案数。

所以只需要用一个 fi,jf_{i, j}fi,j​ 来记录左边 iii 层，右边 jjj 层的方案总数。

考虑顺序。堆这种结构，从下往上是比较方便的，因为这样我们可以不断合并。于是我们就得到了一个转移：

fi,j=fi,j−1×X+fi−1,j×Yf_{i, j} = f_{i, j - 1} \times X + f_{i - 1, j} \times Y fi,j​=fi,j−1​×X+fi−1,j​×Y

那我问你，XXX 和 YYY 是什么啊？

如果将 iii 放到最下面，这个点的权值显然要小于 jjj。所以就可以在所有 iii 子树和 jjj 子树中选，那么选到 i>ji > ji>j 的概率就应该是 Y=2i−12i+2j−2Y = \frac{2^i-1}{2^i+2^j-2}Y=2i+2j−22i−1​。XXX 同理。

The Lost Array

题意

对于序列 ana_nan​, 令 numinum_inumi​ 表示数字 iii 出现的次数, 统计满足条件的序列个数:

  • 1≤ai≤m1 \leq a_i \leq m1≤ai​≤m
  • 给定一些 xi,yix_i, y_ixi​,yi​, 满足 axi≠yia_{x_i} \neq y_iaxi​​=yi​
  • 给定一些 xi,yix_i, y_ixi​,yi​, 满足 numxi≠yinum_{x_i} \neq y_inumxi​​=yi​.
  • 将 numnumnum 排序后, 得到一给定序列 BBB.

n≤16,m≤30n\le 16, m\le 30n≤16,m≤30

解析

首先，考虑一个暴力 DP：设 fi,S,Tf_{i, S, T}fi,S,T​，iii 表示值域从 111 到 iii，SSS 哪些位置已经被填过了，TTT 记录剩下的所有可行的 num\text{num}num 的可重集。

接下来注意到 TTT 会非常小：由于有一大堆 x,yx, yx,y 的限制，同时 ∑Ti≤n\sum T_i \le n∑Ti​≤n，于是暴搜一下发现只有 727272 种可能的情况。这样的话这一维就可以当常数了。因此 iii 的转移是 O(m)O(m)O(m) 的，SSS 是 O(n3n)O(n3^n)O(n3n) 的（因为最终要枚举子集）。

考虑如何压掉 SSS。其实 x,yx, yx,y 的限制并不需要 SSS 的参与，我们可以自己就规避开，那么 SSS 唯一的作用就是限制不能重复填一个数。

那么可以考虑容斥。令 f(S)f(S)f(S) 表示恰好填满 SSS 的方案（选的值和位置双射），而 g(S)g(S)g(S) 表示在 SSS 中可以重复地选的方案（选值任意映射）。

那么 f(S)f(S)f(S) 和 g(S)g(S)g(S) 可以通过互相推导得到：

g(S)=∑T⊆Sf(T)g(S)=\sum_{T \subseteq S} f(T) g(S)=T⊆S∑​f(T)

f(S)=∑T⊆Sg(T)×(−1)∣S∣−∣T∣f(S)=\sum_{T \subseteq S} g(T) \times (-1)^{|S|-|T|} f(S)=T⊆S∑​g(T)×(−1)∣S∣−∣T∣

这样的话，我们就可以把这一维扔掉了，时间复杂度 O(2nnm72)O(2^nnm72)O(2nnm72)。这个 trick 就是把这种双射问题用容斥做掉，然后就可以减少复杂度。

最优化问题

Wonderful Jump

性质 1

每一次跳跃一定在两端取最小值。

证明显然。不然你拆成两段显然更优。

性质 2

考虑如果一步一步跳，跳 LLL 的代价应该是 O(nL)O(nL)O(nL)。

而假如我们一次跳过去呢？其代价应当为 mL2mL^2mL2。其中 mmm 表示最小值。

也就是说，要想让一步跳跃更优，我们有如下不等式：

mL2≤nLmL^2 \le nL mL2≤nL

约分得到：

mL≤nmL \le n mL≤n

所以这样两个东西乘在一块，我们就可以根号分治了。

当 L≤nL \le \sqrt nL≤n​ 时，我们直接暴力即可。

当 m≤nm \le \sqrt nm≤n​ 时，我们又需要讨论两种情况：

  1. mmm 在后面取到。此时，能对答案有贡献的一部分就是从第一个比 mmm 小的数到 mmm。考虑每一个这样的数，它们的贡献区间应该是无交的（因为前面的都算过了，而这个区间如果跨过前面就一定会比拆开劣。）。那么每种最小值的长度都是 nnn，总共不超过 n\sqrt nn​ 种，总共复杂度是 O(nn)O(n\sqrt n)O(nn​)。
  2. mmm 在后面取到。显然这是后缀最小值，最多变化 O(n)O(\sqrt n)O(n​) 次，拼在一起复杂度不变。

然后就完了。

Speed Dial

首先考虑要建一棵字典树。发现快速拨号就是一些前缀，那么就可以在字典树上表示成一个点，然后问题就转化成了所有点到最近的染色的祖先的路径长度和。

如果从内向外考虑，记录有多少点没有连接到染色点的话，转移似乎非常构思。那么我们换一个思路，记录某个节点到上方的第一个染色点的距离——当这个距离等于 000 的时候，就是说这个位置应当被染色。

也就是说，我们设计的状态大概是这样的：fi,j,df_{i, j, d}fi,j,d​ 表示以 iii 为根的子树中选 jjj 个，距离上面的关键点为 ddd 的方案。转移也就比较好想了：

fu,j,d=min⁡{fu,j−x,d+fv,x,d+1}f_{u, j, d} = \min\{f_{u, j - x, d} + f_{v, x, d + 1}\} fu,j,d​=min{fu,j−x,d​+fv,x,d+1​}

为什么这样比较好做呢？因为考虑到这个问题的难点就在于有一些没有连接到染色点的节点。如果记录内部的节点，整个就乱套了（因为这样很难分配），但是在外面就好多了，因为在外面的话，向下转移的时候情况是确定的。

Red Black Tree

假设将一个点变成红色的代价为 cu,0c_{u, 0}cu,0​，反之为 cu,1c_{u, 1}cu,1​

首先不难发现一个 O(n2)O(n^2)O(n2) 的 DP：

fu,x=min⁡{cu,t+∑fv,x−t}f_{u, x} = \min\{c_{u, t} + \sum f_{v, x - t}\} fu,x​=min{cu,t​+∑fv,x−t​}

闵可夫斯基和是一种解决凸函数的上述卷积形式的有力工具。

ccc 显然是凸函数，fff 则可以用归纳法证明：两个凸函数的卷积仍然是一个凸函数。

那么 fu,xf_{u, x}fu,x​ 的最小值就相当于把差分数组的 <0< 0<0 的部分全都加起来，再加上 fu,0f_{u, 0}fu,0​。

接下来考虑求和操作。发现这个操作可以直接变成把差分数组的前缀合并。显然，fu,xf_{u, x}fu,x​ 中，xxx 的取值范围是它所有叶子节点中深度最小的那个。这样的话，合并的复杂度不会超过线性。（就算合并到最长的，复杂度也最多是线性的。）

这样的话，我们可以把 ccc 的差分直接扔到 ∑f\sum f∑f 的差分数组里。如果用 set 的话是 O(nlog⁡n)O(n \log n)O(nlogn) 的，但是如果用 vector 分别存储记录正数和负数，再记录 000 的个数的话，可以做到 O(n)O(n)O(n)。
