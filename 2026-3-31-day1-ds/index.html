支配对类

这类问题通常是给定一个结构，要求查询区间上两个位置的某种贡献的最小值或者最大值。如果直接找的话复杂度是 O(n2)O(n^2)O(n2)，但是通常可以找到一种方式，只保留其中的一些点对（通常是 O(nlog⁡n)O(n \log n)O(nlogn)），这样的话整个问题的复杂度就降下来了。保留下来的部分就叫做这些点对的一个支配对。

Souvenirs

这个恐怕是最经典的题目了。

考虑对于一个位置 iii，有多少位置可以和 iii 产生贡献？

首先 i+1i + 1i+1 一定可以和 iii 产生贡献。

对于 i+2i + 2i+2，先不考虑 ai+2<aia_{i + 2} < a_iai+2​<ai​ 的情况。在前面两个值的中间分开，假如说 ai+2>ai+ai+12a_{i + 2} > \frac{a_i + a_{i + 1}}2ai+2​>2ai​+ai+1​​，就说明这个点应该是和 i+1i + 1i+1 产生贡献是更优的。

反之则会和 iii 产生贡献，这样的话这条分界线就向下移动一半。因此只需要每次找到后面第一个在区间内的值。这可以通过主席树维护。当然还需要考虑小于的情况，这可以反过来计算。最终会得到 O(nlog⁡n)O(n\log n)O(nlogn) 个点对。

将这些点对放到平面上，每次相当于查询一个矩形内的点的最小值。再观察一下不难发现其实由于下面一半是没有点的，所以可以改成计算右下方的点，这可以扫描线做。

Minimum Xor On Segment

和上一个题的思路差不多。

假设 jjj 一定会和 iii 产生贡献。考虑一个 kkk 位置会不会产生贡献：

首先将三个数表示成 0/10/10/1 串，求出它们的 LCP。这段所有的贡献都没有区别。

接下来考察下一位。假如说 iii 和 jjj 在这一位是相同的，那么新的这个一定没用（因为前面是 LCP，所以这一位一定不同）。因此只有可能是 iii 和 jjj 不同且 kkk 和 iii 这一位相同。这样 LCP 就往后+1，而 LCP 最多只会到 log⁡V\log VlogV，所以同样可以求出简单的支配对。

对复杂维分治

如果一个题有两维限制，一维相对于另一维不好用数据结构维护，就可以尝试分治这一维，从而取消这个限制。

常见于树分治。

[Ynoi2004] rpmtdq

树这一维限制非常扫码，因为没什么好的方式来维护一个区间里面的点对。而如果能够找出这个结构的支配对，那么区间这一维就变成套板子了。

考虑边分治（实际上点分治也是一样的，但是边分治好看一些）。对于一个点 uuu，定义它到当前层的中心的距离为 dud_udu​。那么两点之间的距离就可以用 du+dvd_u + d_vdu​+dv​ 来表示（同子树内不用管）。这样考虑对于 uuu 这个节点，只考虑它对面那棵子树中 dv≤dud_v \le d_udv​≤du​ 的点（反过来的可以反着再做一次），这样假如有两个点 jjj 和 kkk，jjj 和 kkk 的距离是小于 dj+dk≤2×dud_j + d_k \le 2 \times d_udj​+dk​≤2×du​ 的，因此其实除了前驱后继都可以直接扔了。这样就又转化回了数点问题。

TEST_199

这个和最开始那个的区别就是多了个 bbb。

假设存在一种求支配对的方式，可以让不管 bbb 怎么变，这个点对都保留了所有的可能解，那么将一个点的颜色定义为 bi⊕bjb_i \oplus b_jbi​⊕bj​ 就变成了矩形点反色、矩形查白点。这可以 KDT 做，加上一些随机优化复杂度就比较正确了。

接下来就是如何找到这种支配集。显然之前的那种是不可行的，但是经过一些推理可以得到结论：把原先的点数量扩大一倍就可以。

依然是考虑哪些点可以对 iii 产生贡献。注意此时的贡献是对于任何一种染色方式都可行的。

后面至少有两个点都是一定会产生贡献的，因为不能确定颜色。接下来考虑第三个点（假设这三个点分别为 jjj，kkk，lll）：由于 iii 至少与其中一个同色，所以新的这个点就分别和 jjj 和 kkk 的中点比较，和之前是一样的。

K-D Tree

感觉这个结构非常有意思，既可以搞一些复杂度固定的，也可以搞一些随机复杂度的内容。

复杂度

范围查询

也就是矩形查询。

这个复杂度是可以证明的。

考虑一条垂线会和 KDT 上的点有哪些情况。

  1. 和这个节点完全无交。对复杂度没有贡献。
  2. 不完全有交。这种情况下，考虑这个节点的四个孙子，他们将这个节点所对应的点等分成了四份，而一条垂线最多会到达两个子节点。当前节点的查询都是 O(1)O(1)O(1) 的，这样就可以写出递归复杂度的表达式：
T(n)=2T(n4)+O(1)=O(n) T(n) = 2T(\frac n4) + O(1) = O(\sqrt n) T(n)=2T(4n​)+O(1)=O(n​)

这样，考虑对于一个查询的矩形，其左右两个边界的查询复杂度都是垂线的复杂度，总的复杂度一定就是 O(n)O(\sqrt n)O(n​)。

这可以扩展到高维，方法类似。

最近邻查询

这是个典型的乱搞。

「Cfz Round 7」终极 AVX2 硬件指令提速

看标题就知道这个题和标题没关系。

在数据随机的情况下，K-D Tree 可以解决和半平面有关的问题，复杂度相对优秀。

这个题最大的问题是，区间并集带权求和很难维护。一个性质是，点和半平面的问题是对偶的，也就是说可以将点和半平面互换，然后就可以考虑对每个原来的点，计算他对多少个查询有贡献，也就是转化以后对一个半平面上的点进行累加。

这样就可以扫描线了。扫描的是每个原先的点对应的区间的值域。也就是说要实现下面的事情：

  • 将一个半平面内的所有点的 cnt±1cnt \pm 1cnt±1。
  • 将全局 cnt=0cnt = 0cnt=0 的点的 sumsumsum 加上 xxx。

第一个操作容易实现，只需要打个标记就可以了。第二个操作看起来不容易做，但是实际上只需要查询全局和，所以其实每次只需要找 cnt=0cnt=0cnt=0 有多少个点就可以了。
