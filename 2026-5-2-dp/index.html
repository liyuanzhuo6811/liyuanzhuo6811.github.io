SDOI2026二轮省集 DP（其实全是数数）

发表于2026-05-02|更新于2026-05-02|2026二轮省集

Tree Cutting

考虑对有根树做，把每个点作为根的方案数都算出来。显然会算重，但是考虑对于一个连通块，重复的数量就相当于是以每个节点作为根都数了一遍，那么最后直接除以 sks_ksk​ 就可以了。

那么现在就相当于是以一个点为根，要求所有有祖孙关系的边都是先删掉深度大的，统计方案。问题在于多个子树可以合并，因此需要记录一个目前已经达成的 kkk 的集合，那么转移的时候就每次子集枚举然后合并一个儿子进来即可。同时，考虑当前集合的 kkk 的 highbit⁡\operatorname{highbit}highbit 可能是根节点的父边，此时需要满足剩下的部分正好是 shs_hsh​。

由于是树上合并，不难用前后缀合并的方式实现换根。（recollator还在追我）

Prediction

考虑钦定前缀最大值序列 qqq，那么与 qiq_iqi​ 不合法的数一定是被扔到 qi+1q_{i + 1}qi+1​ 后面去了，因此从大到小填前缀最大值，设 lil_ili​ 表示最左边与 aia_iai​ 不合法的位置，那么从 lqil_{q_i}lqi​​ 到 lqi+1l_{q_{i + 1}}lqi+1​​ 这一块都得扔到后面，容易算排列数。

那么就用一个 DP 记录这个就可以了。

Multiple of Three Cycles

已经成环的位置没有意义，因为肯定不会再断开，而链的具体长度也是没有用处的，只需要关心长度对 333 的余数就可以了。

那么问题就转化为了求有多少种方案数能够将这些链合并成一些长度为 333 的环，设 f(a,b,c)f(a, b, c)f(a,b,c) 分别表示 1/2/01/2/01/2/0 分别数量下最终都能够合并到环的方案数。

由于可以每次把一个 000 链挂到任意一个链上并且不改变余数（挂到自己上相当于成环），我们有 f(a,b,c)=(a+b+c)f(a,b,c−1)f(a, b, c) = (a + b + c)f(a, b, c - 1)f(a,b,c)=(a+b+c)f(a,b,c−1)。于是实际上只要知道了 c=0c = 0c=0 的情况就可以得到整个序列，那么实际上就只需要令 g(a,b)=f(a,b,0)g(a, b) = f(a, b, 0)g(a,b)=f(a,b,0)，算出来就可以了。

考虑这个是可以递推的，分别考虑把一条长度为 111 和长度为 222 的链挂起来，可以得到下面两种递推方式：

g(a,b)=b(a+b−1)g(a−1,b−1)+(a−1)g(a−2,b+1) g(a,b)=b(a+b−1)g(a−1,b−1)+(a−1)g(a−2,b+1) g(a,b)=b(a+b−1)g(a−1,b−1)+(a−1)g(a−2,b+1) g(a,b)=a(a+b−1)g(a−1,b−1)+(b−1)g(a+1,b−2) g(a,b)=a(a+b−1)g(a−1,b−1)+(b−1)g(a+1,b−2) g(a,b)=a(a+b−1)g(a−1,b−1)+(b−1)g(a+1,b−2)

不过这个还是很难递推，不过考虑实际上并不是每种状态都有用，观察一下，实际上如果从后往前的顺序，也就相当于是每次断掉一条链（当然如果操作以后没有发生变化那就不做改变），那么假如上一次查询了 (u,v)(u, v)(u,v)，那么下一次可能查询的就只有 (u,v)(u, v)(u,v)（断环或者分出来一段倍数），(u+1,v+1)(u+1, v+1)(u+1,v+1)（断掉一条整链），(u+2,v−1)(u + 2, v - 1)(u+2,v−1)（从中间切开一个 222），(u−1,v+2)(u - 1, v + 2)(u−1,v+2)（从中间切开一个 111）。

那么考虑记录 XXX 和 YYY 分别表示当前的 g(a,b)g(a, b)g(a,b) 和 g(a+1,b+1)g(a + 1, b + 1)g(a+1,b+1)，考虑这样递推式里面就只有两个未知数，可以对于四种情况都能够解出新的 XXX 和 YYY。这样就做完了。

这期真神了。

Boxes and Balls

把交换操作等价为把一个 111 通过临项交换平移，那么就可以记录一个暴力的状态，设 f(i,j,k)f(i, j, k)f(i,j,k) 表示前 iii 个位置用了 jjj 步，前面有 kkk 个 111 的方案数。

转移是平凡的，考虑把第 k+1k + 1k+1 个 111 挪过来即可。问题是这个复杂度略显爆炸。考虑实际上这个状态很卡不满，要把第一个 111 挪进来至少一步操作，第二个至少两步操作（因为第一个位置已经放过了），这样全都平移过来的话，实际上对于 jjj 次操作只能弄出来根号个 111，所以与原序列的差不会超过 O(k)O(\sqrt k)O(k​) 级别，于是做完了。

Jellyfish and Hack

不难注意到实际上给定的 lim⁡\operatorname{lim}lim 如果超过 O(n2)O(n^2)O(n2) 就根本没用（快排最差也就到 O(n2)O(n^2)O(n2) 的复杂度），所以就可以通过暴力记录一个长度为 xxx 的序列需要进行 yyy 次操作的方案数 f(x,y)f(x, y)f(x,y)。然后这个东西转移的时候需要同时枚举分割点和操作次数，可以获得高达 O(n6)O(n^6)O(n6) 的复杂度，显然炸了。

如果固定分割点，就相当于是个背包的卷积形式，除了没法用 NTT 以外都挺好的。

考虑写出生成函数的形式，不难写出生成函数的递推形式，那么就可以把这个生成函数看做一个多项式，对他暴力求出值，然后拉格朗日插值就可以得到最终的系数了。（这真的是人类吗）

然后拉插的时候，每一次只有一项是不一样的，所以就每次做个多项式除法，相当于背包撤销的操作就可以了（recollator能不能别追我了）

Jellyfish and OEIS

首先可以容斥，就是钦定一些区间是个排列。

然后发现就算给个带着集合的 DP 都非常困难，因为根本没法转移。考虑容斥的时候，某些位置实际上存在一个对称位置，他们之间容斥系数相反且可以构成双射（也就是说能够产生完全相同的情况），那么他俩就消掉了，没有产生新的贡献。

于是找一下这种情况，发现实际上只要有两个区间是有交的就一定会被消掉，那么现在只需要找到所有包含当前位置的方案。

那么可以通过一个区间 DP，钦定当前区间必须选，然后就根本没法转移了。考虑问题在于需要空余位置，因此再记一个 g(l,r,k)g(l, r, k)g(l,r,k) 表示加上空了几个位置，这样就可以转移了。
